How it works
  1. Each submit creates a batch/v1 Job labelled for Kueue (queue-name + priority-class), suspended so Kueue controls scheduling.
  2. Kueue admits Jobs into a ClusterQueue with a fixed 6-CPU quota. Work that doesn't fit waits as pending.
  3. When a higher-priority job needs room, Kueue preempts (evicts) a running lower-priority job — watch a row flip to PREEMPTED.
  4. Admitted pods run on auto-provisioned Spot nodes; the board shows which node and for how long.
